# رامش‌کالا — فروشگاه اینترنتی لوازم خانگی و نظم‌دهنده‌ها

اسکلت اولیه پروژه با Next.js 14 (App Router)، Tailwind CSS و Prisma.

## راه‌اندازی روی سیستم خودت

1. Node.js نسخه ۱۸ به بالا نصب باشد.
2. نصب پکیج‌ها:
   ```bash
   npm install
   ```
3. کپی فایل env:
   ```bash
   cp .env.example .env
   ```
4. ساخت دیتابیس (SQLite برای شروع) و اجرای Prisma:
   ```bash
   npx prisma migrate dev --name init
   npm run prisma:seed
   ```
5. اجرای پروژه:
   ```bash
   npm run dev
   ```
   سپس آدرس http://localhost:3000 را در مرورگر باز کن.

پنل مدیریت در آدرس http://localhost:3000/admin در دسترس است.

## رفع مشکل «نمی‌توانم محصول را حذف کنم»

اگه محصولی توی یه سفارش (حتی سفارش آزمایشی) خریده شده باشه، از این به بعد می‌تونی بدون مشکل حذفش کنی؛
توی اون سفارش قدیمی به‌جای اسم محصول می‌نویسه «محصول حذف‌شده» ولی خود سفارش پاک نمی‌شه.
چون یه رابطه توی دیتابیس تغییر کرده، این دستور رو یه بار دیگه بزن:
```bash
npx prisma migrate dev --name product-optional-in-orderitem
```

## اضافه کردن دسته‌بندی‌های بیشتر (مشابه ساختار سایت‌های مشابه)

یه اسکریپت اضافه شده که چند دسته‌بندی متداول (کمد و لباس، بانکه و ظروف شیشه‌ای، آرایشی، کشو، رخت‌خواب، خودرو، جاکفشی)
رو بدون پاک کردن اطلاعات فعلیت اضافه می‌کنه:
```bash
npm run seed:categories
```
هر چند بار که بخوای می‌تونی این دستور رو بزنی، دسته‌بندی‌های تکراری رد می‌شن.

## بازطراحی الهام‌گرفته از گالری سعید

صفحه اصلی، هدر و فوتر بازطراحی شدن: بنر تمام‌عرض با عکس، دسته‌بندی‌های عکس‌دار، بخش تب‌دار «ترین‌ها»،
بخش «فروش ویژه» با نشان درصد تخفیف، و دکمه واتساپ شناور.

- برای فعال کردن دکمه واتساپ، مقدار `NEXT_PUBLIC_WHATSAPP_NUMBER` رو توی `.env` پر کن.
- چون فیلد `image` به دسته‌بندی‌ها اضافه شده، یه بار دیگه این دستور رو بزن:
  ```bash
  npx prisma migrate dev --name add_category_image
  ```
- از پنل ادمین (`/admin/categories`) برای هر دسته‌بندی یه لینک عکس اضافه کن تا توی صفحه اصلی قشنگ نمایش داده بشه.

## راه‌اندازی پیامک کد تایید (OTP) با کاوه‌نگار

1. توی سایت kavenegar.com ثبت‌نام کن (یه مقدار اعتبار رایگان هدیه می‌گیری برای تست).
2. از پنلش، بخش «اعتبارسنجی / تعریف الگوی اعتبار سنجی» یک الگو (Template) بساز — مثلاً به اسم `verify` — و توی متنش حتماً از `%token%` استفاده کن (جایی که کد باید قرار بگیره).
3. کلید API رو از پنل کپی کن و توی `.env` بذار:
   ```
   KAVENEGAR_API_KEY="کلید-خودت"
   KAVENEGAR_TEMPLATE="verify"
   ```
4. اگه هنوز این مرحله رو انجام ندادی، اشکالی نداره — تا وقتی `KAVENEGAR_API_KEY` خالی باشه، سایت به‌جای ارسال پیامک واقعی، کد تایید رو توی همون ترمینال سیاه چاپ می‌کنه (`⚠️ کد تایید برای 09... : 12345`) تا بتونی کل فرآیند ورود رو بدون هزینه تست کنی.
5. چون یه جدول جدید (`OtpCode`) به دیتابیس اضافه شده، باید یه بار دیگه این دستور رو بزنی:
   ```bash
   npx prisma migrate dev --name add_otp
   ```

## راه‌اندازی ورود به پنل مدیریت

از این به بعد `/admin` رمزدار شده. فایل `.env` رو باز کن و این سه خط رو با مقادیر دلخواه خودت پر کن:
```
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="یک رمز قوی انتخاب کن"
AUTH_SECRET="یک رشته تصادفی طولانی، هرچی دلت خواست"
```
بعد برو به `http://localhost:3000/admin-login` و با همون نام کاربری/رمز وارد شو.

## راه‌اندازی درگاه پرداخت زرین‌پال

1. توی فایل `.env` این سه خط رو (که از `.env.example` اومدن) چک کن:
   ```
   ZARINPAL_MERCHANT_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
   ZARINPAL_SANDBOX="true"
   NEXT_PUBLIC_BASE_URL="http://localhost:3000"
   ```
2. تا وقتی مرچنت آیدی واقعی از zarinpal.com نگرفتی، `ZARINPAL_SANDBOX` رو روی `"true"` نگه دار — تو این حالت پول واقعی جابه‌جا نمی‌شه و می‌تونی کل فرآیند خرید رو تست کنی.
3. برای گرفتن مرچنت آیدی واقعی: توی zarinpal.com ثبت‌نام کن، یک درگاه پرداخت بساز، مرچنت آیدی رو کپی کن و جای مقدار `ZARINPAL_MERCHANT_ID` بذار. بعد `ZARINPAL_SANDBOX` رو `"false"` کن.
4. چون یک فیلد جدید (`authority`, `refId`) به مدل سفارش اضافه شده، باید یک بار دیگه این دستور رو بزنی:
   ```bash
   npx prisma migrate dev --name add_payment_fields
   ```

## نکته مهم درباره فونت

فایل‌های فونت IRANSansX در `public/fonts/` قرار داده نشده‌اند (چون فونت دارای مجوز استفاده است).
فایل‌های `IRANSansX-Regular.woff2`, `IRANSansX-Medium.woff2`, `IRANSansX-Bold.woff2` را خودت
از منبع مجاز (مثلاً fontiran.com یا خرید لایسنس) دانلود کن و در همان مسیر قرار بده. تا قبل از آن،
سایت با فونت جایگزین (Tahoma) نمایش داده می‌شود.

## وضعیت فعلی پروژه

### آماده و کارکردن با دیتای نمونه (lib/data.js):
- صفحه اصلی، لیست محصولات با فیلتر دسته‌بندی، جزئیات محصول
- سبد خرید (state سمت کاربر، ذخیره در sessionStorage)
- تسویه حساب (فرم آدرس — هنوز به درگاه پرداخت واقعی وصل نیست)
- ورود با موبایل (فرم OTP — هنوز به سرویس پیامک واقعی وصل نیست)
- پروفایل کاربر، تماس با ما، درباره ما، سوالات متداول
- پنل مدیریت: داشبورد، محصولات، دسته‌بندی‌ها، سفارش‌ها، کاربران، تنظیمات (نمایشی — هنوز به دیتابیس متصل نیست)

### آماده شده ولی نیاز به اتصال دارد:
- `prisma/schema.prisma`: مدل‌های Category، Product، User، Address، Order، OrderItem
- `prisma/seed.js`: پر کردن دیتابیس با همان دیتای نمونه
- `lib/prisma.js`: کلاینت Prisma آماده استفاده

### قدم بعدی برای تبدیل به نسخه واقعی:
1. صفحات و پنل ادمین را از `lib/data.js` (دیتای نمونه) به کوئری‌های Prisma واقعی وصل کن (`lib/prisma.js`).
2. API Routeهای مربوط به سفارش، سبد خرید سمت سرور و احراز هویت را در `app/api/...` بساز.
3. اتصال به یک درگاه پرداخت ایرانی (مثل زرین‌پال) در صفحه checkout.
4. اتصال به یک سرویس پیامک (مثل کاوه‌نگار یا ملی‌پیامک) برای ارسال کد OTP در صفحه login.
5. آپلود واقعی تصاویر محصولات (مثلاً با یک storage مثل S3 یا لیارا آبجکت استوریج).

## ساختار پوشه‌ها

```
app/                → صفحات (App Router)
  admin/             → پنل مدیریت
  products/[slug]/   → جزئیات محصول
  cart/ checkout/ login/ profile/ contact/ about/ faq/
components/          → کامپوننت‌های مشترک (Header, Footer, ProductCard, CartProvider)
lib/                 → دیتای نمونه + کلاینت Prisma
prisma/              → schema و seed
```
